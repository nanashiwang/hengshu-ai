# 衡术 Hengshu · 完整闭环体验手册

> 一页走通：发现 → 下载/安装 → 本地运行 → 兼容回流 → 术值/留存 → 求术。
> 地址：dev `http://localhost:3000`，容器 `http://localhost:8787`。本手册以 `:3000` 为例。

---

## 0. 启动

```bash
docker compose up -d                 # postgres(5433) / redis(6380)
cp .env.example .env                 # 填 PAYLOAD_SECRET；MODEL_GATEWAY_* 可留空(走 mock)
npm install && npm run dev           # http://localhost:3000
npm run seed                         # 分类 + 20 官方 Skill；开发无管理员时打印一次性随机密码
npm run migrate:spec-v1              # Spec v1（幂等）
npm run artifacts:backfill           # 冻结制品（幂等）
npm run seed:rules                   # 术值规则（幂等）
```

开发提示：首访 `/admin` 创建首个管理员最稳；若用 `npm run seed` 自动建首管，开发会打印一次性随机密码。生产必须显式设置 `SEED_ADMIN_PASSWORD`，且默认不创建固定 `WELCOME1`。

## 1. Web 端（发现 / 试用 / 下载）

1. 打开 `/` → `/skills` 浏览市场（PT 风格表，可分类/排序/搜索）。
2. 进 `/skills/xhs-title-generator`：
   - **下载 Skill**（主操作，YAML/JSON，含 `integrity.checksum`）。
   - **在线试用** → 表单填写运行；切「多模型对比」勾 2–3 个模型并排看成本/耗时/JSON 率。
   - 右下角 🔒 是该版本冻结快照的 checksum。
3. 登录后可**收藏**、发**评论评分**。
4. 浅/深色主题：右上角 ☀️/🌙。

> 注册：开发环境如 seed 创建了 `WELCOME1` 可用它体验；生产请使用真实邀请链。

## 2. 本地 Runner（登录 / 安装 / 离线运行 / 兼容回流）

```bash
# 登录：终端显示设备码 → 浏览器打开提示的 /device?code=... 授权
node runner/hengshu.mjs login --hub http://localhost:3000
node runner/hengshu.mjs whoami                       # 确认归属

# 安装到本地 ~/.hengshu/skills（自动校验 checksum + ed25519 签名，无效拒装）
node runner/hengshu.mjs install xhs-title-generator
node runner/hengshu.mjs list

# 运行（已装则离线读本地）。--report 回传兼容指标（不含输入/输出原文）
node runner/hengshu.mjs run xhs-title-generator \
  --endpoint http://localhost:11434/v1 --model qwen2.5 \
  --in topic=秋季护肤 --in style=专业 --report
#   匿名贡献加 --anon；用自有网关：--endpoint <gw>/v1 --model <m> --key sk-xxx

# 更新 / 体检
node runner/hengshu.mjs outdated
node runner/hengshu.mjs update
node runner/hengshu.mjs doctor --endpoint http://localhost:11434/v1 --model qwen2.5
```

回传后刷新 `/skills/xhs-title-generator`：「**本地模型兼容报告**」表出现该模型的成功率/JSON 率/耗时，标题显示 **LocalScore**；下载区显示 `✓签名`。

> 信任模型：兼容报告默认**仅展示**；管理员把某 Runner 标为 **verified**（后台 RunnerClients.trustedLevel）后，其具名报告才**计术值**（+3/次、日上限 20），兼容表对应行标 `✓N`。

## 3. 社区（求术闭环）

1. `/bounties` →「+ 发布悬赏」填术值赏金（发布即从你**冻结**对应术值）。
2. 另一账号在悬赏详情 `/bounties/[id]` 点**认领** → 填交付的 Skill slug **提交**。
3. 发布人点**验收并发放赏金** → 冻结术值释放给接单人；或**取消**全额退还。

## 4. 我的控制台 `/console`

左侧字标题导航 + 右侧内容区。**个人**：概览 / 已安装 Skill（含**待更新**）/ Runner 实例 / 运行记录 / 术值流水 / 收藏 / 邀请码。
**管理**（仅 admin/reviewer）：按后台分组折叠为左侧小标题 **Skill 内容 / 成员管理 / 审核治理 / 系统设置**，点开某集合即在内容区内嵌该后台表。
URL 跟随字标题：`/console/installs`、`/console/admin/skills`、`/console/admin/site-settings`…（旧 `/me` 自动重定向到 `/console`）。

---

## 5. 管理后台

两种入口，同一套 Payload Admin：
- **控制台内嵌** `/console/admin/<集合>`：左侧「管理」分组导航，内容区只呈现该集合的表（已隐藏后台自带左导航）。
- **独立后台** `/admin`：完整后台；每页右上「在新标签打开 ↗」可跳转。

仅 admin/reviewer 可进。分组：**Skill 内容 / 成员管理 / 审核治理 / 系统设置**。
> 「第一个注册的用户自动成为超级管理员」；后续在后台调角色、术值规则、审核等。

## 速查：核心命令

| 动作 | 命令 |
|---|---|
| 登录 | `node runner/hengshu.mjs login --hub <url>` |
| 安装 | `node runner/hengshu.mjs install <slug>` |
| 运行+回传 | `node runner/hengshu.mjs run <slug> --endpoint <ep> --model <m> --report` |
| 下载 manifest | `GET /v1/skills/<slug>/manifest?format=yaml` |
| 在线运行 | `POST /v1/skills/<slug>/run`（登录 Cookie） |
| 多模型对比 | `POST /v1/skills/<slug>/compare {input, models[]}` |
