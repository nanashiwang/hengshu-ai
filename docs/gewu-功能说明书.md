# 格物 用户功能说明

> 这是一份给普通用户看的功能导览。你可以把它当成 格物 的地图：先知道每个入口能解决什么问题，再决定从哪里开始用。

格物 围绕一件事展开：让 AI Skill 像软件包一样拥有身份、版本、签名、兼容证据、失败记录和企业治理能力。用户不用为了一个 Skill 更换模型；格物 负责帮你判断它是否可信、是否适配你已有的模型/网关/Runner/企业环境。

---

## 首页

> 入口：/

首页负责把用户带进 格物。第一次打开时，你会看到平台定位、后台 Starter Pack 配置的必备 Skill 快速入口、推荐 Skill、失败知识库入口和验签入口。

首页适合三类人：新用户先了解平台，老用户快速回到常用区域，创作者查看当前有哪些 Skill 正在被推荐。

你通常会从这里先跑必备 Skill，或进入 Skill 市场、模型榜、失败库、公开验签页，登录后进入自己的控制台。运营可以在后台调整 Starter Pack 的顺序、推荐理由和公开默认示例，让新用户更快跑出第一条有价值结果。

---

## Skill 市场

> 入口：/skills

Skill 市场是找能力的入口，不是单纯 Prompt 货架。这里展示已经发布的 AI Skill，每个 Skill 都对应一种可复用任务能力，并逐步带上 Passport、兼容证据、失败记录和达标证书；列表里会直接显示 Passport 可信分，方便先看证据再试用。

用户可以搜索关键词，也可以按必备、分类、最新、热门、评分、运行表现等方式筛选。列表行动列可直接进入试跑页或查看该 Skill 的私人台账。新用户建议先用“必备 Skill”筛选，按“看 Passport → 默认输入试跑 → 回控制台看台账/重跑”的路径快速跑通一个低风险任务。

如果你现在只有一个任务想法，比如“帮我写小红书标题”或“把文章整理成 JSON”，优先来这里找现成 Skill。

---

## Skill 详情页

> 入口：/skills/[slug]

详情页回答一个问题：这个 Skill 值不值得用。

页面会展示 Skill 的用途、作者、分类、版本、输入字段、输出格式、Skill Passport、黄金样例基准、兼容证据、失败记录、达标证书、成功率、成本和耗时。右侧提供下载、在线试用、收藏、Fork 等操作；Passport 区域可导出证据包 JSON，便于采购或审计离线归档。

重点看四块内容：Passport 告诉你是否可信，黄金样例基准告诉你系统测试是否达标，兼容表现告诉你你的模型能不能稳跑，失败记录告诉你可能踩什么坑。兼容矩阵里的每个模型都能继续跳到模型画像、该 Skill×模型的失败库和 Adapter API。

---

## 在线试用

> 入口：/skills/[slug]/run

在线试用适合先验货。用户不需要安装 Runner，直接在网页填写输入参数，选择模型或运行模式，就能看到输出结果。

试跑页会先提示 Passport 可信分，并提供 Passport、Contract、达标证书和证据验签入口。在线 API 可携带模型供应商和模型版本，方便把运行结果回流到对应 ModelProfile、失败库和 Adapter 版本链路。这样用户不是盲跑一个 Prompt，而是在知道身份、能力契约和证据状态后再运行。

试跑后可以看到本次运行是否成功、用了哪个模型、耗时多少、成本多少、输出格式是否稳定。觉得好用再收藏、下载或长期使用。

第一次接触某个 Skill 时，建议先在线试用一次，再决定是否安装到本地。

---

## 多模型对比

> 入口：Skill 在线试用页

多模型对比用来解决“这个 Skill 是否适配我已有模型”的问题。同一个输入可以交给多个模型运行，结果会放在一起比较，并沉淀到可筛选导出的私人运行台账。

用户主要看输出质量、格式是否稳定、响应速度、成本和失败原因。便宜模型如果已经够用，就没必要每次都用更贵的模型；如果失败，可以进入失败库查看修复方向。

这个模块适合模型选择困难、成本敏感，或需要为某个固定任务确定默认模型的用户。

---

## 模型中立榜

> 入口：/models

模型中立榜展示模型在全站真实运行中的表现。它不是泛泛的模型能力榜，而是更关注任务执行里的成功率、格式稳定性、延迟、官方报价和成本效率；也可以按模型名、Provider 和画像状态筛选。

用户可以按成本效率、质量、速度、价格排序。样本不足的模型会显示为积累中，避免少量数据造成误导。

当你准备长期使用某个模型，或者想知道某个模型在平台里表现是否稳定，可以先看这里。每行都能跳到模型画像 API、该模型相关的失败库和 Adapter API，方便从“模型是否漂移”继续追到“哪些 Skill 需要 Adapter、哪些补丁已经有效”。

---

## 可信发现榜

> 入口：/rank

可信发现榜用于发现高质量 Skill 和活跃贡献者。它适合没有明确关键词、但想看看平台上什么内容值得试的人。

Skill 可信榜会同时展示可信分、成功率、运行次数和 Passport 可信分；页面会解释基础可信分、可信证据加权和饱和防刷规则，每个 Skill 可展开查看“为什么排这里”和采用建议；有 Passport 的条目可以直接打开公开证据 API。贡献榜用于发现持续维护 Skill、提交反馈和参与社区建设较多的用户。

可信发现榜的作用不是制造热闹，而是帮用户更快找到可靠内容；样本少的 Skill 可以试用，但不建议只凭排名采购。

---

## 失败知识库

> 入口：/failures

失败知识库记录模型和 Skill 运行时常见的问题，例如 JSON 输出失败、内容截断、格式不稳定、请求失败等。

公开页面会展示错误症状、可能原因、来源分布、证据验签入口和修复方向；审核员可以单条或批量确认归因、根因分类和复验覆盖；页面和公开 API 都可按 Skill、Profile、输入档、模型版本、来源和错误类型筛选。失败画像的 profileKey 已细分到 Skill × 输入档 × 错误类型 × 模型版本，避免把不同版本的漂移混成同一类问题。每个失败画像都能继续跳到模型画像、Adapter API 和证据验签，方便从“为什么失败”继续追到“哪个模型漂移、哪个补丁有效”。它的闭环是“发现失败模式 → 生成 Adapter 草稿 → 审核批准后自动复验 → 写回 lift 和证据快照”。具体修复模板会转成 Adapter 补丁草稿，只允许该 Skill 作者、审核员或管理员查看和生成，公开页不暴露补丁正文。

当你发现某个模型总是跑不稳，或者同一个错误反复出现，先来这里查原因。

---

## 本地 Runner

> 入口：命令行 runner/gewu.mjs

Runner 是 格物 的本地命令行工具。它让用户可以把 Skill 安装到自己的环境，在本地运行，并选择是否把兼容结果回传给平台。

常见动作包括登录、安装、查看列表、运行、更新、体检和回传报告。用户也可以使用自己的模型接口和 API Key 来跑 Skill。

如果你已经有自己的模型 API，或者希望把 Skill 当成长期工具使用，Runner 是更适合的方式。

---

## 设备授权

> 入口：/device

设备授权用于让命令行 Runner 安全登录账号。终端会显示设备码，用户在网页确认后，本地 Runner 就会和账号绑定。

绑定后的设备会出现在控制台的 Runner 实例里。以后如果设备不用了，也可以在控制台撤销。

这个流程主要服务于第一次使用 Runner 的用户。

---

## 我的控制台

> 入口：/console

控制台是用户自己的工作台。登录后，和自己相关的内容都会集中到这里。

你可以查看概览、通知、我的作品、发布 Skill、已安装 Skill、Runner 实例、私人台账、贡献流水、Credit 兑换、收藏、邀请码和个人设置。

如果首页和市场是“发现内容”，控制台就是“管理自己的使用记录和资产”。概览页会突出私人运行台账：总运行、成功、格式有效和换模型重跑，让用户快速看到自己的长期使用资产。

---

## 私人台账

> 入口：/console/runs

私人台账保存用户每次使用 Skill 的结果。它会记录使用的 Skill、模型、输入、输出、成本、耗时、是否成功、格式是否有效，以及本次模型策略带来的成本估算差异。

页面支持按 Skill、模型、路由方式、成功状态、格式状态和重跑来源筛选；导出账本指标或本人输入/输出时，会沿用当前筛选条件。当前页也可以批量换模型重跑，适合把一组历史任务快速迁移到新模型上做成本和稳定性对比。

记录里的“换模型重跑”很重要：同一个任务可以换另一个模型或模型版本再跑一次，也可以批量选择当前页历史运行统一重跑；新结果会写入 `rerunOf` 血缘，批量接口只返回运行摘要，不回显输入/输出。失败记录会直接提供带版本的模型画像和失败库入口，用户可以继续追查是模型漂移、输入规模、输出格式，还是需要 Adapter 修复。

这部分会逐渐变成用户自己的 AI 工作台账，用得越多，越能看出哪些模型适合自己的任务。

---

## BYOK 设置

> 入口：/console/settings

BYOK 是 Bring Your Own Key，意思是用户可以绑定自己的模型 API Key，用自己的额度运行 Skill。

绑定后，用户可以在平台代付和自有 Key 之间选择。自有 Key 更适合已经有模型额度、希望自己控制模型来源和成本的用户。

普通用户只需要理解一句话：如果你已经有模型 API Key，可以填到这里，用自己的 Key 跑 Skill。

---

## 收藏与订阅

> 入口：Skill 详情页、/console/favorites

收藏用于保存常用或感兴趣的 Skill。收藏后，可以在控制台快速找回，也能接收相关更新提醒。

适合长期使用某个写作、总结、分析或格式化 Skill 的用户。常用 Skill 不需要每次重新搜索，直接从收藏列表进入即可。

---

## 评论与评分

> 入口：Skill 详情页

评论和评分用于记录真实使用体验。用户可以给 Skill 打分、写评价、提交失败案例或反馈兼容问题。

这些反馈会帮助其他用户判断 Skill 是否好用，也能帮助创作者改进版本。

如果一个 Skill 跑得很好，或者某个模型上明显不稳定，都建议留下反馈。

---

## 求术悬赏

> 入口：/bounties

求术是“我想要一个 Skill，但现在市场里没有”的解决方式。用户发布需求，设置贡献值悬赏，创作者认领并提交 Skill，发布人验收后发放贡献值。

这个模块把用户需求变成新的 Skill 供给，而不是一次性答案。发布需求时建议写清输入、输出格式和验收标准，方便创作者交付可版本化、可签名、可继续沉淀 Passport/兼容证据的 Skill。

适合找不到现成 Skill，但知道自己想要什么功能的用户。

---

## 创作者发布

> 入口：/console/skills/new

创作者可以在这里发布自己的 Skill。发布页会按“上传包 → 生成 Contract → 刷新 Passport → 适配维护”解释闭环；发布时需要填写标题、简介、分类，并上传包含 manifest、输入输出 schema、示例、权限和推荐模型的 Skill 包。运营侧也可以用来源清单批量导入 GitHub README、Claude Skill、GPTs 配置或 格物 Skill 包，先进入待审 Imported Skill。

发布后，Skill 会进入市场，用户可以试用、收藏、评论和反馈。待审阶段作者可以预览 Contract、Passport 和证书；审核发布后 Passport 会刷新为当前状态，证书才可能从预备状态变成正式达标。创作者可以在“我的作品”查看每个 Skill 的发布状态、Contract 状态、Passport 可信分、证书预览和失败库入口，再根据真实运行反馈持续更新版本或生成 Adapter 草稿。

适合能沉淀任务模板、结构化输出或 AI 工作流，并希望把能力分享给其他用户的人。

---

## 贡献值

> 入口：/console/contributions

贡献值用于记录用户对内容和数据的贡献。发布优质 Skill、提交有效兼容报告、完成求术任务、贡献有效反馈，都可能获得贡献值。

贡献值可以用于发布求术悬赏，也可以在规则允许时兑换部分 credit。它体现的是信誉和贡献，不是现金，不能提现，也不用于随机抽奖。

---

## credit

> 入口：/console/exchange

credit 是平台内用于模型调用的额度。当用户使用平台代付模型运行 Skill 时，会消耗 credit。

如果用户绑定自己的 API Key，主要消耗的是自己的模型额度；如果使用平台模型，则会消耗平台 credit。

用户可以在控制台查看余额、兑换状态和运行消耗。

## 通知中心

> 入口：/console/notifications

通知中心提醒用户和自己相关的变化，比如收藏的 Skill 更新、已安装 Skill 有新版本、求术状态变化、评论审核消息、系统提醒等。

对于经常使用 Skill 或参与创作的用户，通知中心可以避免错过版本更新和悬赏进展。

## 邀请码

> 入口：/console/invites

邀请码用于邀请新用户加入平台，也用于记录邀请关系。

用户可以查看自己的邀请码，复制邀请链接，并把链接发给其他人注册。平台采用邀请机制时，新用户通常会通过这个入口进入。

## 公开验签

> 入口：/verify

公开验签用于复核平台关键证据：分数快照、绑定 Contract 的 Skill 达标证书、Passport/FailureCase/Adapter 证据快照，以及外锚 JSONL/manifest。页面按这四类证据解释验签路径，用户或合作方可以查看签名状态、哈希、签名时间；达标证书可从详情页通过 `/verify?certificateUrl=...` 自动载入并验签；证据快照表单可输入 targetType 和 targetId，也可从详情页或失败库通过 `/verify?targetType=...&targetId=...` 直达，查看公开脱敏的 targetSummary、payloadHash 和签名状态；外锚校验可复核链头、文件哈希、可信发布目标、外部时间戳回执和可信等级；管理员/审核员也可以调用已配置 TSA 服务签发真实时间戳回执。证据快照验签需要已知对象，不提供匿名全量枚举。

它的作用是让榜单、证据和达标证书更可复核：历史记录不应该被无痕改写，采购或企业审计也可以独立验签。达标证书会绑定能力契约摘要；验签页面也会展示该 Contract 摘要，避免只证明某次评分、不证明 Skill 当时的输入输出约束与权限边界。如果证书还是预备状态，页面会解释原因，比如缺少黄金样例基准、manifest 未签名、Passport 还不是当前状态，或证据快照验签未通过。

普通用户不一定每天使用这个页面，但如果关心平台数据可信度，可以来这里查看。

## 开发者文档

开发者文档说明 Skill 如何作为接口被调用，如何读取 Contract/Passport/证书，以及如何接入 OpenAI 兼容 endpoint 或本地 Runner。

开发者可以查看运行接口示例、输入输出结构、BYOK/endpoint 说明和后台入口。普通用户如果只是网页使用，可以先不用看这部分。

## 企业 Registry

> 入口：/console/enterprise

企业 Registry 用来把 Skill 变成企业可治理资产，而不是让员工各自找 Prompt。管理员可以为已批准或待准入的 Skill 绑定策略包、身份策略和审计边界；OIDC SSO 校验通过后会直接签发平台登录会话。

页面按“治理总览 → 批准 Skill → 绑定策略 → 私有评测 → 留审计 → 查失败库”的闭环组织：总览会先聚合 Registry 状态、准入待办、SSO/SCIM 状态、成员、近期审计和失败模式；也可以打开组织内 Passport/证书状态、导出企业准入证据包、批量查看哪些准入因 Contract/Passport/证书漂移需要重审，或用企业自己的私有样例跑组织内评测。私有评测会校验模型白名单和审计策略，只写运行台账与企业审计，不进入公开榜、公开 Passport 或公开兼容报告，也不在响应里回显输入输出原文。企业证书接口会额外给出证书状态摘要，方便采购或审计判断它是正式达标、预备状态还是未达标。企业审计只记录治理元数据、模型版本和规模档，不暴露员工输入输出原文。

---

## 管理后台

> 入口：/admin 或 /console/admin

管理后台面向管理员、审核员和运营人员。普通用户通常不需要进入。

后台可以管理用户、Skill、版本、制品、分类、私人台账、兼容报告、悬赏、邀请码、贡献规则、评论、举报、媒体和系统设置。

平台内容审核、用户治理、数据排查和系统配置都在这里完成。

---

## 部署设置

> 入口：/console/admin/deployment-settings

部署设置是管理员配置平台运行参数的地方，包括站点地址、OpenAI 兼容 endpoint / New API 履约配置、manifest 签名、备份确认和生产运行相关开关。

普通用户不需要关心这一页。平台从本地测试切换到 NAS、服务器或公网生产环境时，管理员需要检查这里。

## 常见使用路径

> 入口：按你的身份选择

普通用户：进入首页，先跑必备 Skill；再打开 Skill 市场，选择一个 Skill，看 Passport/兼容证据/达标证书，在线试用后收藏或查看私人台账。

本地 Runner 用户：先在终端登录，完成设备授权，安装 Skill，本地运行，再选择是否回传兼容报告。

创作者：进入控制台，发布 Skill，填写输入字段、能力契约和输出格式，发布版本，然后根据兼容回流、失败库和可筛选的 Adapter 建议继续维护。

求术用户：进入悬赏区，发布需求和贡献值悬赏，等待创作者提交，验收后发放贡献值。

企业管理员：进入企业 Registry，审批 Skill，绑定策略包和身份策略，导出审计 CSV，查看组织内失败知识库。

管理员：进入后台，检查部署设置，管理 Skill 和用户，处理举报，查看运行、兼容、证据快照、企业 Registry 和验签配置。
